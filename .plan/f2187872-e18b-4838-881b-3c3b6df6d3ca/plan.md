# Settings 组件修复计划

## 问题分析

用户反馈"添加模型"按钮只能点击一次，无法连续添加多个模型。

## 根本原因

当前代码逻辑：
1. 点击"添加模型"按钮 → `setIsAdding(true)` 显示表单
2. 表单通过 `{isAdding && (...)}` 条件渲染
3. 点击"添加"按钮后，`handleAddModel` 只清空了 `newModelName`，没有关闭表单
4. **问题**：表单一直显示，但用户可能误以为按钮失效

实际上代码逻辑是正确的，表单应该保持打开。但用户反馈"只能点击一次"，可能是：
- AnimatePresence 动画问题导致表单没有正确渲染
- 或者用户期望的是每次点击"添加模型"按钮都能看到反馈

## 解决方案

移除 `isAdding` 状态和 AnimatePresence，让添加表单**始终显示**，用户可以持续添加模型。

## 修改内容

1. 删除 `isAdding` 状态
2. 删除 `handleCloseAddForm` 函数
3. 删除 AnimatePresence 包裹
4. 让添加表单始终可见
5. 删除"添加模型"按钮（因为表单始终显示）
